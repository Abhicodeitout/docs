# Revoking Sessions

We use the `logout` mutation (refer to the [backend setup](./backend-setup)) to log the user out. Note that in this example if a session does not already exist the `logout` mutation will throw an error.

```tsx
import Session from "supertokens-node/recipe/session";

const resolvers = {
    // ...

    Mutation: {
        // ...

        logout: async (_: any, args: any, contextValue: any) => {
            let session = await Session.getSession(contextValue.req, contextValue.res);
            await session.revokeSession();
            return "Session revoked";
        }
    },
}
```

:::note
In the code above we read `req` and `res` from `contextValue`, refer to the [backend setup](./backend-setup) to see how this is being added to the graphql context
:::
