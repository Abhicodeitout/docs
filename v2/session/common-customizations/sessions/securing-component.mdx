# Securing a website route / component
You may want to prevent rendering of some part of UI, or a route, when
there's no session.

<FrontendSDKTabs>
<TabItem value="reactjs">

To achieve that, pass `requireAuth={true}` as a prop to `SessionAuth`:

<Question
    question="Do you use react-router-dom?">
<Answer title="Yes">

```jsx 
// highlight-next-line 
import {SessionAuth} from "supertokens-auth-react/recipe/session";

class App extends React.Components {
    render() {
        return (
            <Router>
                <Switch>
                    {...}
                    // highlight-start
                    <Route path="/dashboard">
                        <SessionAuth requireAuth={true} redirectToLogin={() => {/* redirect */}}>
                            // TODO: Components that require to be protected by authentication
                            <MyDashboardComponent />
                        </SessionAuth>
                    </Route>
                    // highlight-end

                    // Other components without authentication
                    {...}
                </Switch>
            </Router>
        );
    }
}
```

</Answer>
<Answer title="No">

```jsx
const App = () => {
    return (
        <SessionAuth requireAuth={true} redirectToLogin={() => {/* redirect */}}>
            {/* MyComponent will be displayed only if session exists */}
            <MyComponent />
        </SessionAuth>
    );
}
```

</Answer>
</Question>

:::important
If you pass `requireAuth={true}`, you have to pass function to `redirectToLogin` prop,
which will redirect the user to where they might log in.
:::

Read more about `SessionAuth` and `requireAuth` in [its API reference](/docs/auth-react/session/session-auth).
