## `supertokens_python.init`

You need to call the `supertokens_python.init` function to initialise SuperTokens. You can provide it the following config:

```python
from supertokens_python import init

init({
	'framework': '...'
	'supertokens': {
		'connection_uri': '...',
		'api_key': '...' # optional
	},
	'app_info': {
		'app_name': '...',
		'website_domain': '...',
		'api_domain': '...',
		'api_base_path': '...', # optional
		'website_base_path': '...', # optional
		'api_gateway_path': '...', # optional
	},
	'recipe_list': []
})
```

## Parameters
### `framework`
- Framework in which this library being used. Currently supported values is `fastapi`.
### `supertokens`
- `connection_uri`
  - The url link to supertokens core. If you are using Supertokens Saas service, you can find the connectionURI on your app's dashboard.
- `api_key` (Optional)
  - Use this to authenticate the backend APIs with the supertokens core
  - Default: `""`
### ``app_info``
- You can learn more about the `AppInfo` object in the recipe guides.
### ``recipe_list``
- List of Recipes you want to use. You need to pass atleast one Recipe to ths array. For more info on Recipes, checkout this [link](/docs/community/recipes) or the other sections in this docs.
