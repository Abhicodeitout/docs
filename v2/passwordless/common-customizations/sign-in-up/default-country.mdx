# Setting default country for phone inputs

If your users are allowed to log in using both email addresses and phone numbers, then the login form accepts both as an input, creating some ambiguous situations. 
If the input is well-formed, it will pass either the email or phone number validation function, and the form can call the API. However, if it passes neither, it means that it was malformed, but we cannot be sure if the user intended it as a phone number or an email. To decide, we check if more than half of the characters are appropriate for a phone number. If they are, we assume it's intended to be one and switch the input field to provide better UX while entering phone numbers.
Entering your phone number in a local format is a simple mistake. You can make it easier for users to correct it in two ways:
- Providing a `defaultCountry`
- Providing the `guessInternationPhoneNumberFromInputPhoneNumber` callback.

## Setting the default country

If you set `defaultCountry` to an appropriate two-letter ISO country code the SDK will use `libphonenumber-js` to parse the entered phone number and reformat it as an international number to prefill the phone number input after the switch selects the country.

<FrontendSDKTabs>
<TabItem value="reactjs">

```jsx
SuperTokens.init({
    appInfo: {...},
    recipeList: [
        Passwordless.init({
            // highlight-start
            signInUpFeature: {
               /*
               * Must be a two-letter ISO country code (e.g.: "US")
               */
                defaultCountry: "HU",
            }
            // highlight-end
        }),
        Session.init({...})
    ]
});
```

## Providing the callback

By providing the `guessInternationPhoneNumberFromInputPhoneNumber` you can take full control of how this heuristic works. The callback can be a sync an async function that returns:
- A string that will be used to prefill the phone number input
- `undefined` if the UI should not change to the phone number input
