# Customising the email sent

## The default email
- From: no-reply@supertokens.io, but the user will see your app name
- Subject: `Sign in instructions`

<img width="450px" src="/img/passwordless/email-verify-email.png" />


This is achieved by calling an API provided by us (`https://api.supertokens.io`). The backend SDK calls our API with the magic link, the app name, and the email of the end-user.

:::tip security
- We do not log / store any of this information on our servers.
- For production use, we recommend that you use the feature to send emails yourself (see below section) using your own domain. This will make it easier for end-users to trust the email (since it's coming from your domain, and not from @supertokens.io)
:::

## Send a custom email

You can take full control of sending an email verification email by providing the `deliverLoginInfo` function during `init` function call:

<BackendSDKTabs>
<TabItem value="nodejs">

```jsx
supertokens.init({
    supertokens: {...},
    appInfo: {...},
    recipeList: [
        ^{recipeNameCapitalLetters}.init({
            // highlight-start
            loginFeature: {
                deliverLoginInfo: async (email, loginUrl, loginCode) => {
                    let {id, email} = user;
                    // TODO:
                }
            } 
            // highlight-end
        }),
        Session.init()
    ]
});
```
</TabItem>
<TabItem value="go">

```go
supertokens.Init(supertokens.TypeInput{
    RecipeList: []supertokens.Recipe{
        ^{codeImportRecipeName}.Init(&^{goModelName}.TypeInput{
            // highlight-start
            LoginFeature: &^{goModelName}.TypeInputLoginFeature{
                DeliverLoginInfo: func(email string, loginUrl string, loginCode string) {
                    // TODO: Create and send verification email
                },
            },
            // highlight-end
        }),
    },
})
```
</TabItem>
</BackendSDKTabs>

- You can get the user's email via the `user` input param.
- Your email should direct the user to open the `loginUrl` link. This link is a full URL with the sign-in token. Clicking this link will sign the user in on the browser that opened the link.
- If you enabled the OTP logins, your email should also include the `loginCode`. Your users could copy and paste this code to sing into your application. To see how to enable this flow, please see the quick-setup [guide](../quick-setup/frontend).
- Any errors thrown from this function will be ignored.

:::important
When using this callback, you must send the email yourself.
:::
