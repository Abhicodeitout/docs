# Embed Sign In / Up form in a page 

## Step 1: Disable default implementation

This will prevent SuperTokens from displaying the default login UI in the `/auth` page.

<PasswordlessFrontendForm>

<FrontendSDKTabs>
<TabItem value="reactjs">
<ConditionalSection conditions={{ contactMethod: "PHONE"}}>

```jsx
SuperTokens.init({
    appInfo: {...},
    recipeList: [
        Passwordless.init({
            mobileForm: {
                // highlight-next-line
                disableDefaultImplementation: true,
                (...)
            },
            (...)
        }),
        (...)
    ]
});
```

</ConditionalSection>
<ConditionalSection conditions={{ contactMethod: "EMAIL"}}>

```jsx
SuperTokens.init({
    appInfo: {...},
    recipeList: [
        Passwordless.init({
            emailForm: {
                // highlight-next-line
                disableDefaultImplementation: true,
                (...)
            },
            (...)
        }),
        (...)
    ]
});
```

</ConditionalSection>
</TabItem>
</FrontendSDKTabs>
</PasswordlessFrontendForm>

If you navigate to `/auth`, you should not see the widget anymore.

## Step 2: Render the component yourself

For example if you would like to add the Sign-up / Sign-in widget at the very end of a landing page, before the footer, simply import the `SignInUp` component and render it:

<FrontendSDKTabs>
<TabItem value="reactjs">

```jsx

// highlight-next-line
import { SignInUp } from 'supertokens-auth-react/recipe/passwordless'; 

class MyLandingPage extends React.Component {
    render() {
        return (
            <div>
                <Header />
                <FirstSection />
                <AnotherSection />

                (...)
                
                // highlight-next-line
                <SignInUp />
                <Footer />
            </div>
        )
    }
}
```
</TabItem>
</FrontendSDKTabs>

## Step 3: Chaning the website path for the login UI (optional)

The default path for this is component is `/{websiteBasePath}/`.

If you are displaying this at some custom path, then you need add additional config on frontend:

<FrontendSDKTabs>
<TabItem value="reactjs">

```jsx
SuperTokens.init({
    recipeList: [
        Passwordless.init({
            //highlight-start

            // The user will be taken to the custom path when then need to login.
            getRedirectionURL: async (context) => {
                if (context.action === "SIGN_IN_AND_UP") {
                    return "/custom-login-path;
                };
            }
            //highlight-end
        })
    ]
})
```
</TabItem>
</FrontendSDKTabs>
