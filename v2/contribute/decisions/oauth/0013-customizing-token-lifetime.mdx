# Token lifetimes should default to values set in the core config

<DecisionHeader status="proposed" lastUpdate="2023-05-11" created="2023-05-11" deciders={["rishabhpoddar", "porcellus"]} proposedBy={["porcellus"]} />

## Context and Problem Statement

Token lifetimes can be controlled in multiple way.

## Considered Options

* Always set by the backend
* Overrideable by backend - defaulting to core config setting

## Decision Outcome

Chosen option: Overrideable by backend - defaulting to core config setting

* Customizable
* More convenient when creating M2M tokens in PostMan

## Pros and Cons of the Options

### Always set by the backend

<ArgumentPro> Customizable </ArgumentPro>
<ArgumentCon> Has to be set in each request even when manually creating token </ArgumentCon>

### Overrideable by backend - defaulting to core config setting

<ArgumentPro> Customizable </ArgumentPro>
<ArgumentPro> More convenient when creating M2M tokens in PostMan </ArgumentPro>
<ArgumentCon> Requires new config options in Core </ArgumentCon>
