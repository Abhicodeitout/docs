# Refresh tokens are single-use only

<DecisionHeader status="proposed" lastUpdate="2023-05-11" created="2023-05-11" deciders={["rishabhpoddar", "porcellus"]} proposedBy={["porcellus"]} />

## Context and Problem Statement

Refresh token rotation is optional in OAuth2.0 but required in 2.1

## Considered Options

* No refresh token rotation
* Always use rotating refresh tokens

## Decision Outcome

Chosen option: Always use rotating refresh tokens

* Fits the newer security best practices

## Pros and Cons of the Options

### No refresh token rotation

<ArgumentPro> Simpler usage </ArgumentPro>
<ArgumentCon> Goes against new security best practices </ArgumentCon>

### Always use rotating refresh tokens

<ArgumentPro> Fits the newer security best practices </ArgumentPro>
<ArgumentPro> We already do something very similar for our normal sessions </ArgumentPro>
<ArgumentCon> There are some edge-cases that make this harder to use (i.e.: connection interruption during token refresh) </ArgumentCon>
