# Redirect to the standard login route if required

<DecisionHeader status="proposed" lastUpdate="2023-05-11" created="2023-05-11" deciders={["rishabhpoddar", "porcellus"]} proposedBy={["porcellus"]} />

## Context and Problem Statement

When the user redirected to the app during the OAuth2 sign-in process, there are a few routes we can choose to provide.

## Considered Options

* Merge functionality into existing auth page
* Add new route for OAuth2 specific authentication that embeds the sign in/up components
* Add new route that redirects to default auth page

## Decision Outcome

Chosen option: Add new route that redirects to default auth page

* Nicely separates responsibilities

## Pros and Cons of the Options

### Merge functionality into existing auth page

<ArgumentPro> No redirection/navigation </ArgumentPro>
<ArgumentCon> Erodes the separation of recipes </ArgumentCon>
<ArgumentCon> Complicates implementation/testing </ArgumentCon>

### Add new route for OAuth2 specific authentication that embeds the sign-in/up components

<ArgumentPro> No redirection/navigation </ArgumentPro>
<ArgumentCon> Duplicates functionality </ArgumentCon>

### Add new route that redirects to default auth page

<ArgumentPro> Nicely separates responsibilities </ArgumentPro>
<ArgumentCon> Requires redirection </ArgumentCon>
