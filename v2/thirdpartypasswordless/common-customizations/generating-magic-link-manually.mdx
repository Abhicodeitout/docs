# Manually generating a link

You can use our backend SDK to generate magic links as shown below:

<BackendSDKTabs>
<TabItem value="nodejs">

```tsx
import ^{recipeNameCapitalLetters} from "supertokens-node/recipe/^{codeImportRecipeName}";

async function createMagicLink(email: string) {
    const magicLink = await ^{recipeNameCapitalLetters}.createMagicLink({email});

    console.log(magicLink);
}
```

</TabItem>
<TabItem value="go">

```go
import (
    "github.com/supertokens/supertokens-golang/recipe/^{codeImportRecipeName}"
)

func main() {
    email := "..."

    magicLink, err := ^{codeImportRecipeName}.CreateMagicLink(email)
    if err != nil {
        // handle error
    }

    fmt.Println(magicLink)
}
```

</TabItem>
<TabItem value="python">

from supertokens_python.recipe.^{codeImportRecipeName}.asyncio import create_magic_link

```

```

<CustomAdmonition type="multi-tenancy">

For a multi tenant setup, you can pass in the user's tenantId to the `createMagicLink` function as well. The generated link will use the configured `websiteDomain` from the `appInfo` object (in `supertokens.init`), however, you can change the domain of the generated link to match that of the tenant ID.

This will ensure that when the user clicks on the link and signs up, they sign up to the tenant you want to give them access to.

</CustomAdmonition>
