# Social / SSO login

You can see the default UI when you visit the `websiteBasePath` (`/auth?rid=^{codeImportRecipeName}` by default) on your website.

## Default social login button UI

<img src="/img/^{codeImportRecipeName}-sign-in-custom-provider.png" alt="Sign in form UI for social login" />

## Callback page UI
When the user is navigated back to your application from the third party provider, they will see the following UI

<img src="/img/^{codeImportRecipeName}-thirdparty-callback.png" alt="Callback screen from third party provider." />


## Sign in / up unsuccessful UI

SuperTokens requires that the thirdparty provider gives an email for the user. If that's not the case for a certain provider, the user will see the following UI.

<img src="/img/^{codeImportRecipeName}-no-email.png" alt="No email UI for provider" />

## General error UI

If there are network related errors, or the backend sends a status code >= 300, then we show the following UI. This UI is also shown if there is a similar error in the callback page.

<img src="/img/^{codeImportRecipeName}-general-error.png" alt="Sign in form general error" />
