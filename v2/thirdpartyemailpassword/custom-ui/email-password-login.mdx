# Email Password login

## Sign up form

Call the following API when the user clicks on the sign up button.

<TabItem value="web">

<NpmOrScriptTabs>
<TabItem value="npm">

```tsx
import { emailPasswordSignUp } from "supertokens-web-js/recipe/^{codeImportRecipeName}";

async function signUpClicked(email: string, password: string, name: string) {
    try {
        let response = await emailPasswordSignUp({
            formFields: [{
                id: "email", // required field
                value: email
            }, {
                id: "password", // required field
                value: password
            },
            {
                id: "name", // custom field - the ID can be anything you like
                value: name
            }]
        })

        if (response.status === "FIELD_ERROR") {
            // one of the input formFields failed validaiton
            response.formFields.forEach(formField => {
                if (formField.id === "email") {
                    // Email validation failed (for example incorrect email syntax),
                    // or the email is not unique.
                    window.alert("Invalid email, or the email already exists")
                } else if (formField.id === "password") {
                    // Password validation failed.
                    // Maybe it didn't match the password strength
                    window.alert("Password is weak")
                } else if (formField.id === "name") {
                    // name field validation failed
                    window.alert("name input failed validation");
                }
            })
        } else {
            // sign up successful. The session tokens are automatically handled by
            // the frontend SDK.
            window.location.href = "/homepage"
        }
    } catch (err: any) {
        if (err.isSuperTokensGeneralError === true) {
            // this may be a custom error message sent from the API by you.
            window.alert(err.message);
        } else {
            window.alert("Oops! Something went wrong.");
        }
    }
}
```

The `formFields` input is a key-value array. As seen above, along with the email and password, you can add the user's name or anything else you like as part of the API call and consume it on the backend.

### Checking if email is unique in the sign up form

As a part of the sign up form, you may want to explicitly check that the entered email is unique. Whilst this is already done via the sign up API call, it may be a better UX to warn the user about a non unique email right after they finish typing it.

You can do this by calling the following API

<FrontendCustomUITabs>
<TabItem value="web">

<NpmOrScriptTabs>
<TabItem value="npm">

```tsx
import { doesEmailExist } from "supertokens-web-js/recipe/^{codeImportRecipeName}";

async function checkEmail(email: string) {
    try {
        let response = await doesEmailExist({
            email
        });

        if (response.doesExist) {
            window.alert("Email already exists. Please sign in instead")
        }
    } catch (err: any) {
        if (err.isSuperTokensGeneralError === true) {
            // this may be a custom error message sent from the API by you.
            window.alert(err.message);
        } else {
            window.alert("Oops! Something went wrong.");
        }
    }
}
```
