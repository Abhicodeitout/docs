# Changing OAuth Scopes

If you would like to add additional OAuth Scopes when accessing your third party provider, you can do so by adding them to the config when initializing the backend SDK.

For example if you are using Google as your third party provider, you can add an additional scope as follows:

<BackendSDKTabs>
<TabItem value="nodejs">

```jsx
Supertokens.init({
    supertokens: {...},
    appInfo: {...},
    recipeList: [
        ThirdPartyEmailPassword.init({
            providers: [
                ThirdPartyEmailPassword.Google({
                    clientSecret: "GOOGLE_CLIENT_SECRET",
                    clientId: "GOOGLE_CLIENT_ID",
                    scope: [
                        "scope1",
                        "scope2",
                    ]
                })
            ]
        })
    ]
});
```

You can learn more about customizing third party configs [here](/docs/nodejs/thirdpartyemailpassword/providers/about)

</TabItem>
<TabItem value="go">

```go
func main() {
	supertokens.Init(supertokens.TypeInput{
		RecipeList: []supertokens.Recipe{
			thirdpartyemailpassword.Init(&tpepmodels.TypeInput{
				Providers: []tpmodels.TypeProvider{
					thirdparty.Google(tpmodels.GoogleConfig{
						ClientID:     "GOOGLE_CLIENT_ID",
						ClientSecret: "GOOGLE_CLIENT_SECRET",
						Scope: []string{
							"scope1", "scope2",
						},
					}),
				},
			}),
		},
	})
}
```

You can learn more about customizing third party configs [here](/docs/golang/thirdpartyemailpassword/providers/about)

</TabItem>
