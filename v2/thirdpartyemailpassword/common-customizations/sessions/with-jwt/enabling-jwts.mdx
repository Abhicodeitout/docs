# Enable JWTs with sessions

:::caution
Using JWTs is optional and is only required if you want to integrate with another service that relies on JWTs or if you wanted to integrate with a backend framework that we do not support yet
:::

## Enable JWT feature

<BackendSDKTabs>
<TabItem value="nodejs">

```jsx
let SuperTokens = require("supertokens-node");
let Session = require("supertokens-node/recipe/session");

SuperTokens.init({
    supertokens: {...},
    appInfo: {...},
    recipeList: [
        Session.init({
            //highlight-start
            jwt: {
                enable: true,
            },
            //highlight-end
        })
    ]
});
```

## Using a custom issuer

By default SuperTokens uses your <BackendSDKCasing>apiDomain/auth</BackendSDKCasing> for the issuer URL. To change the path use <BackendSDKCasing>apiBasePath</BackendSDKCasing> when initialising SuperTokens.

In some cases you may need to provide a custom issuer, for example during development you may need to test with external services (like Hasura Cloud). Since the JWKS endpoint is exposed via your backend, JWT verification will fail because the service may not be able to query your local environment. You can expose your local environment to the internet (using ngrok for example), and set a custom issuer URL instead:

<BackendSDKTabs>
<TabItem value="nodejs">

```jsx
let SuperTokens = require("supertokens-node");
let Session = require("supertokens-node/recipe/session");

SuperTokens.init({
    supertokens: {...},
    appInfo: {...},
    recipeList: [
        Session.init({
            //highlight-start
            jwt: {
                enable: true,
                issuer: "https://api.test.com/auth",
            },
            //highlight-end
        })
    ]
});
```
